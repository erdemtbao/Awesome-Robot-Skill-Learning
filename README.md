## Overview

In embodied AI, a *skill* is a reusable, goal-directed behavior that maps perception, context, and task intent to a temporally extended action. It can take the form of a motion primitive, a learned sub-policy, an affordance-conditioned controller, or a language-callable routine such as grasping, navigation, or tool use. Robot skill learning studies how such behaviors are **represented, discovered, imitated, reinforced, composed, transferred, and continually expanded**, so that robots can solve new long-horizon tasks with less data and more reliable real-world execution. This list organizes the literature along those axes, together with the benchmarks, datasets, and tutorials that support the field.

The columns in each table are: **Title**, **Venue**, **Paper**, **Project**, **Code**, **Robot / Dataset / Benchmark**, **Classification**, **Sim**, **Real**, **Skill**, and a one-line **Summary**. Empty fields and `-` mean the information could not be reliably confirmed.

| EBench | Manipulation (mobile / skill diagnosis) | Sim | [Paper](https://arxiv.org/abs/2606.18239) | Simulation benchmark that diagnoses generalist mobile manipulation policies (π0, π0.5, XVLA, InternVLA-A1) across 26 tasks along 5 capability and 4 generalization dimensions, profiling per-atomic-skill strengths beyond a single success rate. |
